Turn Rmarkdown output into Github Pages

If you Google this topic you will find this GitHub and RStudio tutorial, which is quite long.

Here is the short version (i.e., some short supplementary notes):

Name the output html file as index.html and go to repo settings -> GitHub Pages and point the source to where this file is. Also you can change theme just by clicking the buttom.

Github Page

I guess you can only assign the short domain names like liuyanguu.github.io to one repo (in my case, this blog). All the rest pages/site hosted on Github will have the repo name behind: like liuyanguu.github.io/COVerAge_Plot

The “Add welcome page, theme and navigation” part of the GitHub and RStudio tutorial tells you how to turn Github page into site.

  1. Create in the root directory a file like "_site.yml"

Make sure the yaml format is correct.

The following example as in the tutorial (format corrected) creates a navbar with website title “Cars”, home page named “HomePage” (using index.html) and “Page 1” using the “speed-and-distance.html”.

The name of the html files shall match your rmarkdown output.

site example

name: "cars"
output_dir: "docs"
navbar:
  title: "Cars"
  left:
    - text: "HomePage"
      href: index.html
    - text: "Page 1"
      href: speed-and-distance.html
output:
  html_document:
    theme: cosmo
  1. Render site and push to Github
rmarkdown::render_site()

Think the Github page as a one-page site with index.html as the homepage.
